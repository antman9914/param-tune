# MODULE 2: 训练协议分析

## 2.1 读取优化器配置

从 config 文件读取当前的 optimizer 相关设置（optimizer 类型、lr_scheduler 等），用于理解本轮训练的学习动态，为稳定性预检提供上下文。

## 2.2 读取本次实验配置

**若尚无任何训练结果文件（首次运行）**：
- 直接从 config 文件读取当前参数值
- 跳过 2.4 的训练稳定性预检，标注"预检结果：首次运行，无历史数据"
- 继续输出 Module 2 摘要后，进入 Step 3

否则，从 config 文件读取当前实际使用的参数值。

只显示 **tunable_params**（来自 Module 1 的列表）中的参数，以及固定参数中与训练行为相关的关键项（如 optimizer 类型、epochs 等，根据实际 config 内容决定）：

```

  <param_1> : <值>
  <param_2> : <值>
  ...

固定配置（关键项）：
```

## 2.3 训练稳定性预检

从最新训练输出中读取训练初期（第一个 epoch 或前几步）的 loss 值，做以下检查：

- 初始 loss 与理论值的偏差：结合任务类型（`task_type`）和 optimizer 配置，判断是否合理
  - 偏差超过 2 倍量级 → ⚠️ 警告，可能有数据、初始化或 loss scale 问题
- 任意步骤出现 NaN 或 inf → 🚫 停止，报告并等待用户处理
- 多个 epoch 后 loss 完全不变 → ⚠️ 可能学习率为 0 或梯度未传播

预检失败时停止循环，不继续分析。
